# 参数选择说明

## 分类器为什么选 SVM

本项目建议使用 RBF-SVM 作为主分类器，KNN 作为对照分类器。

原因如下：

- 调制识别的特征维度不高，样本量适中，适合 SVM。
- ASK、FSK、BPSK、QPSK、16QAM 在低 SNR 下会发生特征重叠，线性分类边界不够稳。
- RBF 核 SVM 能建立非线性分类边界，低信噪比下通常比规则分类器更稳。
- KNN 简单直观，适合作为对照组，但低 SNR 下更容易受局部样本分布影响。

规则分类器没有作为最终方案，是因为它依赖固定阈值。噪声变化后，幅度、相位和频率特征都会波动，固定阈值不容易在所有 SNR 下通用。

## SNR 为什么选 -10:2:20 dB

最终工程采用：

```matlab
params.snrList = -10:2:20;
```

原因如下：

- `-10 dB`：强噪声场景，用来检验算法抗噪能力。
- `0 dB` 附近：中等噪声场景，可以观察准确率快速上升过程。
- `20 dB`：较理想信道，用来验证特征和分类器在干净信号下是否能正确分类。
- `2 dB` 步长：曲线比较平滑，同时运行时间还能接受。

如果只是课堂演示或快速调试，可以改成：

```matlab
params.snrList = -5:5:20;
```

但最终报告中使用 `-10:2:20` 更完整。

## 采样和帧长度为什么这样设置

```matlab
params.samplesPerSymbol = 8;
params.symbolsPerFrame = 256;
```

每符号 8 个采样点可以较好展示时域波形、频谱和相位变化，同时不会让仿真过慢。

每帧 256 个符号可以让幅度、相位、高阶累积量等统计特征更稳定。如果帧太短，统计特征容易受随机符号序列影响；如果帧太长，训练和测试会变慢。

## 训练样本和测试样本为什么这样设置

```matlab
params.trainFramesPerClassPerSNR = 60;
params.testFramesPerClassPerSNR = 30;
```

每类每个 SNR 使用 60 帧训练、30 帧测试，可以兼顾准确率统计稳定性和运行时间。五类调制、16 个 SNR 点下，总训练样本数为：

```text
5 x 16 x 60 = 4800
```

总测试样本数为：

```text
5 x 16 x 30 = 2400
```

这个规模适合课程设计，不会太小，也不会让 MATLAB 运行过久。
